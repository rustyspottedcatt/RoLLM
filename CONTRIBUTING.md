## Development Setup

Prerequisites: [Aftman](https://github.com/LPGhatguy/aftman), [Wally](https://github.com/UpliftGames/wally).

```sh
# Install toolchain (rojo, wally, selene, stylua, remodel)
aftman install

# Install Wally dependencies (promise, signal)
wally install
```

Build and open in Roblox Studio:

```sh
rojo build default.project.json --output RoLLM.rbxl
# then open RoLLM.rbxl in Studio
```

## Code Style

- **StyLua** enforces formatting. Config is in `.stylua.toml`. Run `stylua src examples tests` to format in place.
- **Selene** enforces lint rules. Config is in `selene.toml`.
- Type all function parameters and return values using the types in `src/lib/types.luau`.
- Use `local` variables at the tightest scope possible, especially inside hot loops.
- Prefer `ipairs` over `pairs` for sequential tables.
- Matrix allocation is expensive — avoid allocating inside inner loops where possible.

## Validation

Run all of these before opening a pull request:

```sh
stylua --check src examples tests
selene src examples tests
rojo sourcemap default.project.json --output sourcemap.json
rojo build default.project.json --output RoLLM.rbxl
```

All four must pass without errors or warnings.

## Testing

The smoke test suite lives in `tests/SmokeTests.server.luau`. It covers:

- Character and BPE tokenization round-trips
- Cross-entropy loss forward pass
- One training epoch without crash
- `predict` and `generate` output shape

To run: build the place and execute it in Roblox Studio's server context. All tests print `PASS` or error loudly on failure.

When adding a new feature, add a corresponding test assertion in the smoke test file.

## Commit Convention

Use the [Conventional Commits](https://www.conventionalcommits.org/) format:

```
<type>(<scope>): <short summary>
```

Common types: `feat`, `fix`, `perf`, `refactor`, `docs`, `test`, `chore`.

Examples:

```
feat(attention): add flash-attention-style score scaling
fix(optimizer): correct Adam bias correction for t=1
perf(linalg): pre-transpose B matrix in mm() for cache locality
docs: expand CONTRIBUTING and API reference
```

## Pull Request Process

1. Fork the repository and create a branch from `main`.
2. Make your changes, keeping each PR focused on one concern.
3. Run the full validation suite (see [Validation](#validation)).
4. Fill in the [pull request template](.github/pull_request_template.md) completely.
5. A maintainer will review within a reasonable timeframe. Expect requests for changes on API-surface or training-correctness PRs.

Do **not** commit:
- `Packages/` (Wally output)
- `*.rbxl` or `*.rbxm` place/model files
- `sourcemap.json` (generated by Rojo CI)
- Editor config files unrelated to the project

## What Not to Contribute

- Large model weights or datasets.
- Dependencies beyond the existing Wally packages without prior discussion.
- Roblox-platform-specific API calls inside core library modules (`src/components/`, `src/lib/`). Keep those pure Luau.
- Breaking changes to the public API (`RoLLM.new`, `trainModel`, `generate`, `generateTemperature`, `predict`, `predictTemperature`, `getParameterCount`) without opening a discussion issue first.
